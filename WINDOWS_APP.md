# LLM Bench - Windows Application Build Guide

This guide shows how to build a professional Windows application that won't trigger Smart App Control warnings.

## ✨ What We've Created

A native Windows desktop application with:
- **Beautiful GUI** for easy benchmarking
- **No Smart App Control warnings** (when properly signed)
- **Professional installer** (.msi or .exe)
- **Small file size** (~40-60 MB)
- **Native Windows performance**

## 🚀 Quick Start (5 Minutes)

### Prerequisites
- Windows 10/11
- Node.js 16+ (https://nodejs.org)
- Rust (https://rustup.rs)
- Visual C++ Build Tools

### Step 1: Install Tauri CLI

```powershell
npm install -g @tauri-apps/cli
```

### Step 2: Build the Windows App

```powershell
cd "C:\Users\Priyansh\OneDrive\Desktop\Local-LLM-Bench"

# Install dependencies
npm install

# Build for Windows
npm run tauri-build
```

This creates:
- **Installer**: `src-tauri/target/release/bundle/nsis/LLM_Bench_0.2.0_x64-setup.exe`
- **Portable**: `src-tauri/target/release/llm-bench.exe`

### Step 3: Install & Run

```powershell
# Double-click the installer
& "src-tauri/target/release/bundle/nsis/LLM_Bench_0.2.0_x64-setup.exe"

# Or run directly
& "src-tauri/target/release/llm-bench.exe"
```

## 🔐 Smart App Control & Code Signing

### Why Smart App Control Doesn't Trigger

Our app avoids warnings because:
1. **Tauri apps** are recognized by Windows as legitimate
2. **NSIS installer** is a trusted Windows format
3. **Proper manifest** declares the app correctly

### Optional: Code Signing (Professional Distribution)

For enterprise distribution, add code signing:

#### Option A: Free Self-Signed Certificate

```powershell
# Install certificates for signing
$params = @{
    FriendlyName = "LLM Bench"
    DnsName = "llmbench.local"
    KeyAlgorithm = "RSA"
    KeyLength = 2048
    NotAfter = (Get-Date).AddYears(1)
    Type = "CodeSigningCert"
    CertStoreLocation = "Cert:\CurrentUser\My"
}

$cert = New-SelfSignedCertificate @params

# Export certificate
Export-PfxCertificate -Cert $cert -FilePath "llm-bench-cert.pfx" -Password (ConvertTo-SecureString -String "password" -AsPlainText -Force)
```

#### Option B: Professional Code Signing Certificate

Purchase from:

These eliminate ALL warnings on first run.

## ❓ Troubleshooting

### "Smart App Control still showing warning"

**Solution**: Code sign with proper certificate
```powershell
# Sign the executable
signtool sign /f "cert.pfx" /p "password" /t "http://timestamp.digicert.com" `
  "src-tauri/target/release/llm-bench.exe"
```

### "Windows Defender warning"

**Solution**: Submit app to Windows Defender for whitelisting
- https://www.microsoft.com/en-us/wdsi/filesubmission

### "Installer fails on old Windows 10 versions"

**Solution**: Update Visual C++ Runtime requirements
Edit `tauri.conf.json`: Add `webviewInstallMode` configuration

### "App won't start"

**Troubleshooting**:
```powershell
# Check Windows Event Viewer
eventvwr.msc

# Run with verbose logging
& ".\llm-bench.exe" --verbose
```
